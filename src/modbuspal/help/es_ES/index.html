Modbus Process Plant Simulator - Guía Operativa

Guía principal para simulación de planta y prácticas MODBUS.

Flujo paso a paso Troubleshooting

1) Estructura de interfaz

  • Menú Project: cargar/guardar/limpiar proyecto.
  • Menú Tools: Master, Scripts, Console, Verificar dependencias.
  • Menú Ayuda: documentación y guías operativas contextuales.
  • Menú Language: cambio de idioma en tiempo real.
  • Menú Salir: cierre con persistencia de sesión.
  • Panel de conexión: Link settings + Run/Learn/Record.

2) Controles de ejecución

  • Run: inicia/detiene el enlace seleccionado.
  • Learn: crea puntos automáticamente en escrituras nuevas.
  • Record: registra tráfico MODBUS para replay.
  • ASCII: habilita framing MODBUS ASCII en serial.

3) Conceptos MODBUS y de proceso

  • Holding Registers: consignas y comandos (R/W).
  • Input Registers: telemetría de proceso (R).
  • Coils: comandos booleanos (R/W).
  • Input Coils: estados y alarmas (R).
  • Bindings: conversión de tipos.
  • Functions: lógica personalizada por esclavo.
Puerto 502: endpoint estándar MODBUS/TCP. Error de bind indica puerto ocupado.
Serial en JDK moderno: RXTX puede quedar limitado; use TCP/IP o Replay cuando sea necesario.
