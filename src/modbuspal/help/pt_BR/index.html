Modbus Process Plant Simulator - Guia Operacional

Guia principal para laboratório de automação e simulação de planta completa com MODBUS.

Fluxo passo a passo Troubleshooting

1) Estrutura da interface

  • Menu Project: carregar/salvar/limpar projeto.
  • Menu Tools: Master, Scripts, Console, Check dependencies.
  • Menu Ajuda: documentação e guia operacional contextual.
  • Menu Language: troca de idioma em tempo real.
  • Menu Sair: encerra com persistência do último projeto.
  • Painel de conexão: Link settings + Run/Learn/Record.

2) Operação dos botões de execução

  • Run: inicia/paralisa o link ativo (TCP/IP, Serial, Replay).
  • Learn: cria pontos automaticamente quando o master escreve endereços novos.
  • Record: grava tráfego MODBUS para auditoria e replay.
  • ASCII: habilita framing serial em MODBUS ASCII.

3) Conceitos de processo e MODBUS

  • Holding Registers: setpoints e comandos (R/W).
  • Input Registers: variáveis de processo (R).
  • Coils: comandos booleanos (R/W).
  • Input Coils: estados e alarmes (R).
  • Bindings: conversões de tipo (SINT16, FLOAT32, etc.).
  • Functions: lógica customizada por escravo.
Porta 502: padrão do MODBUS/TCP. Se houver bind error, outra aplicação já está usando a porta.
Serial em JDK moderno: o driver RXTX depende de configuração legada; se indisponível, use TCP/IP ou Replay.
