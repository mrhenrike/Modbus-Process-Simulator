Modbus Process Plant Simulator - Operational Guide

Main guide for plant-process simulation and MODBUS training workflows.

Step-by-step workflow Troubleshooting

1) Interface structure

  • Project menu: load/save/clear project.
  • Tools menu: Master, Scripts, Console, Check dependencies.
  • Help menu: contextual documentation and operational guides.
  • Language menu: live UI language switch.
  • Exit menu: closes app with workspace persistence.
  • Connection panel: Link settings + Run/Learn/Record.

2) Runtime controls

  • Run: starts/stops selected link (TCP/IP, Serial, Replay).
  • Learn: auto-creates points on unknown master writes.
  • Record: records MODBUS traffic for replay and audit.
  • ASCII: enables MODBUS ASCII framing over serial.

3) Process and MODBUS concepts

  • Holding Registers: setpoints and commands (R/W).
  • Input Registers: process telemetry (R).
  • Coils: boolean commands (R/W).
  • Input Coils: status/alarm bits (R).
  • Bindings: data-type conversion (SINT16, FLOAT32, etc.).
  • Functions: custom per-slave logic.
Port 502: standard MODBUS/TCP endpoint. Bind errors indicate the port is already occupied.
Serial on modern JDK: RXTX may be limited on non-legacy runtimes; use TCP/IP or Replay when needed.
