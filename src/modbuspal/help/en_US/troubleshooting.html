Troubleshooting (en-US)

Back to index

Cannot bind port 502

  • Another process is already bound to port 502.
  • Change TCP port or stop the conflicting process.

RXTX / serial unavailable

  • Modern Java runtimes may not expose java.ext.dirs.
  • When serial is unavailable, use TCP/IP or Replay mode.
  • Use Tools > Check dependencies for quick diagnostics.

Project load issues

  • Validate .xmpp integrity and compatibility.
  • Confirm all referenced scripts exist in expected relative paths.
